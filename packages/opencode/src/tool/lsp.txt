Interact with Language Server Protocol (LSP) servers to get code intelligence features.

# REAL POWER - Code Intelligence Engine

You have DEEP code understanding through LSP:

## Code Navigation
- Jump to definition across entire project
- Find all references to any symbol
- Navigate call hierarchies (who calls whom)
- Explore type hierarchies and implementations

## Code Understanding
- Get instant type information for any symbol
- View documentation and hover info
- See all symbols in a file (functions, classes, variables)
- Search project-wide symbols by name

## Code Analysis
- Find all implementations of an interface
- Trace incoming calls (who uses this function)
- Trace outgoing calls (what does this function call)
- Map dependencies between modules

## Supported operations:
- goToDefinition: Find where a symbol is defined
- findReferences: Find all references to a symbol
- hover: Get hover information (documentation, type info) for a symbol
- documentSymbol: Get all symbols (functions, classes, variables) in a document
- workspaceSymbol: List project-wide symbols matching a query string
- goToImplementation: Find implementations of an interface or abstract method
- prepareCallHierarchy: Get call hierarchy item at a position (functions/methods)
- incomingCalls: Find all functions/methods that call the function at a position
- outgoingCalls: Find all functions/methods called by the function at a position

All operations require:
- filePath: The file to operate on
- line: The line number (1-based, as shown in editors)
- character: The character offset (1-based, as shown in editors)

workspaceSymbol also accepts:
- query: A query string to filter symbols by. Empty string requests all symbols.

For workspaceSymbol, filePath is not sent in the LSP workspace/symbol request. It is used by opencode to select and start the matching LSP server.

Note: LSP servers must be configured for the file type. If no server is available, an error will be returned.
