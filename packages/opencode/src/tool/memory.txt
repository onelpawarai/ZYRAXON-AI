Store and recall information from persistent memory. Use this to remember important facts, user preferences, project decisions, and anything that should survive across sessions.

# REAL POWER - Permanent Memory System

Your memory is IMMORTAL. Store EVERYTHING. Recall ANYTHING. Forget NOTHING.

## Memory Types
- Preferences: User likes/dislikes, coding style, language
- Decisions: Architecture choices, library selections, design patterns
- Bugs: Found issues, root causes, fixes applied
- APIs: Endpoint documentation, auth patterns, rate limits
- Code: Key functions, class structures, module relationships
- Projects: Directory structures, build commands, deploy steps

## Actions

- store: Save information with a key and optional tags
- recall: Retrieve a specific memory by key
- search: Search memories by keyword and tags
- list: List all stored memories

## Memory Strategies
- Store after EVERY meaningful interaction
- Recall BEFORE starting any task
- Search when unsure about past decisions
- Tag memories for easy filtering
- Update memories when information changes

## When to Store
- User expresses a preference
- A bug is found and fixed
- An architecture decision is made
- An API endpoint is discovered
- A build command is verified working
- A project structure is mapped

## When to Recall
- Before answering questions about past work
- When the user references previous discussions
- When building on prior decisions
- When checking if something was already tried

## Example

user: remember that I prefer dark mode
assistant: [calls memory with action=store, key=user_preference_theme, content=User prefers dark mode]

user: what did I say about the database?
assistant: [calls memory with action=search, query=database]
