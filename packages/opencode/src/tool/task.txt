Launch a new agent to handle complex, multistep tasks autonomously.

# REAL POWER — Subagent Army

You can launch UNLIMITED subagents to work in parallel on any task:

## Agent Types
- **explore**: Fast codebase exploration — find files, patterns, structures
- **general**: Multi-step task execution — research, implement, verify
- **vision**: Screen analysis — OCR, UI debugging, visual inspection

## Usage
When using the Task tool, you must specify a subagent_type parameter to select which agent type to use.

When NOT to use the Task tool:
- If you want to read a specific file path, use the Read or Glob tool instead
- If you are searching for a specific class definition like "class Foo", use the Grep tool instead
- If you are searching for code within a specific file or set of 2-3 files, use the Read tool instead
- If no available agent is a good fit for the task, use other tools directly

## Parallel Execution
- Launch MULTIPLE agents in a SINGLE message for maximum speed
- Each agent works independently — no dependencies
- Results arrive as they complete — no waiting
- Example: Launch 3 explore agents to search different parts of the codebase simultaneously

## Agent Delegation Rules
1. Launch multiple agents concurrently whenever possible
2. Once you have delegated work to an agent, do not duplicate that work yourself
3. When the agent is done, it returns a single message back to you
4. Each agent invocation starts with a fresh context unless you provide task_id
5. The agent's outputs should generally be trusted
6. Clearly tell the agent whether you expect it to write code or just do research
7. If the agent description mentions that it should be used proactively, use it without the user asking

## Prompt Engineering
- Be SPECIFIC about what you want the agent to do
- Include exact file paths when known
- Specify expected output format
- Include verification steps
- Set clear success criteria

## Advanced Patterns
- Chain agents: Agent A output → Agent B input
- Parallel agents: Launch 3-5 agents for different aspects
- Resume agents: Use task_id to continue a previous session
- Nested agents: Agent can launch its own sub-agents

## When to Use
- Codebase exploration across multiple directories
- Research tasks requiring web searches
- Complex implementations with multiple components
- Parallel verification of changes
- Code review across many files
