## Ustrukturyzowane Dane WyjЕӣciowe

MoЕјesz zaЕјД…daДҮ ustrukturyzowanych danych wyjЕӣciowych JSON od modelu, okreЕӣlajД…c `format` za pomocД… schematu JSON. Model uЕјyje narzДҷdzia `StructuredOutput`, aby zwrГіciДҮ zweryfikowany kod JSON pasujД…cy do Twojego schematu.

### Podstawowe uЕјycie

```typescript
const result = await client.session.prompt({
  path: { id: sessionId },
  body: {
    parts: [{ type: "text", text: "Research Anthropic and provide company info" }],
    format: {
      type: "json_schema",
      schema: {
        type: "object",
        properties: {
          company: { type: "string", description: "Company name" },
          founded: { type: "number", description: "Year founded" },
          products: {
            type: "array",
            items: { type: "string" },
            description: "Main products",
          },
        },
        required: ["company", "founded"],
      },
    },
  },
})

// Access the structured output
console.log(result.data.info.structured_output)
// { company: "Anthropic", founded: 2021, products: ["Claude", "Claude API"] }
```

### Typy formatu wyjЕӣciowego

| Typ           | Opis                                                                      |
| ------------- | ------------------------------------------------------------------------- |
| `text`        | DomyЕӣlny. Standardowa odpowiedЕә tekstowa (brak ustrukturyzowanych danych) |
| `json_schema` | Zwraca zweryfikowany JSON pasujД…cy do dostarczonego schematu              |

### Format JSON Schema

UЕјywajД…c `type: 'json_schema'`, podaj:

| Pole         | Typ             | Opis                                                         |
| ------------ | --------------- | ------------------------------------------------------------ |
| `type`       | `'json_schema'` | Wymagane. OkreЕӣla tryb schematu JSON                         |
| `schema`     | `object`        | Wymagane. Obiekt JSON Schema definiujД…cy strukturДҷ wyjЕӣciowД… |
| `retryCount` | `number`        | Opcjonalne. Liczba ponownych prГіb walidacji (domyЕӣlnie: 2)   |

### ObsЕӮuga bЕӮДҷdГіw

JeЕӣli model nie wygeneruje prawidЕӮowych ustrukturyzowanych danych wyjЕӣciowych po wszystkich prГіbach, odpowiedЕә bДҷdzie zawieraДҮ `StructuredOutputError`:

```typescript

  console.error("Failed to produce structured output:", result.data.info.error.message)
  console.error("Attempts:", result.data.info.error.retries)
}
```

### Najlepsze praktyki

1. **Podawaj jasne opisy** we wЕӮaЕӣciwoЕӣciach schematu, aby pomГіc modelowi zrozumieДҮ, jakie dane wyodrДҷbniДҮ
2. **UЕјywaj `required`**, aby okreЕӣliДҮ, ktГіre pola muszД… byДҮ obecne
3. **Zachowaj schematy skoncentrowane** - zЕӮoЕјone zagnieЕјdЕјone schematy mogД… byДҮ trudniejsze dla modelu do poprawnego wypeЕӮnienia
4. **Ustaw odpowiedni `retryCount`** - zwiДҷksz dla zЕӮoЕјonych schematГіw, zmniejsz dla prostych
